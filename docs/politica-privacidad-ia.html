Politica de Privacidad IA

Como MIIA maneja, almacena y protege los datos de tus leads y clientes

1. Que datos procesa MIIA

Datos que SI procesa

  • Mensajes de texto de WhatsApp
  • Notas de voz (transcripcion temporal)
  • Imagenes y videos (descripcion temporal)
  • Documentos enviados (resumen temporal)
  • Numero de telefono del lead
  • Nombre del contacto (si lo comparte)
  • Historial de conversacion (ultimos 40 mensajes)

Datos que NO almacena

  • Archivos multimedia originales (se procesan y descartan)
  • Contrasenas o datos bancarios
  • Ubicacion GPS
  • Contactos del telefono del lead
  • Mensajes de grupos de WhatsApp
  • Estados de WhatsApp
  • Conversaciones de otros tenants

2. Almacenamiento y retencion

  • Conversaciones: Se almacenan en Google Cloud Firestore, encriptado en transito y en reposo. Cada tenant tiene datos completamente aislados.
  • Historial activo: MIIA mantiene los ultimos 40 mensajes por conversacion en memoria. Los mas antiguos se descartan automaticamente.
  • Media (audio/imagen/video): Se descarga temporalmente en memoria RAM para transcripcion/descripcion, se procesa y se elimina inmediatamente. No se almacena en disco ni en la nube.
  • Backups: Los usuarios pueden exportar sus datos en formato .miia (encriptado con AES-256). Solo el propietario puede importar su backup.

Importante: MIIA no entrena modelos de IA con los datos de tus conversaciones. Los datos se envian a Gemini/OpenAI/Claude unicamente para generar respuestas en tiempo real, no para fine-tuning.

3. Proveedores de IA

MIIA envia el contenido de los mensajes a uno de estos proveedores para generar respuestas:

  • Google Gemini — Sujeto a la Politica de uso de Google AI. No usa datos de API para entrenar modelos.
  • OpenAI — Sujeto a la Politica de datos de API de OpenAI. Los datos de API no se usan para entrenar.
  • Anthropic (Claude) — Sujeto a la Politica de privacidad de Anthropic. No entrena con datos de API.

4. Derechos del lead

  • Opt-out: Cualquier lead puede enviar "baja", "no molestar", "spam" o "unsubscribe" para que MIIA deje de contactarlo permanentemente.
  • Comando STOP: Detiene todas las comunicaciones automaticas inmediatamente.
  • Eliminacion: El propietario del tenant puede eliminar conversaciones y datos de un lead especifico.

5. Seguridad tecnica

  • Autenticacion via Firebase Auth (OAuth 2.0 + email/password)
  • Sesiones de WhatsApp almacenadas en Firestore con chunks encriptados
  • Comunicacion HTTPS (TLS 1.2+) para todas las APIs
  • Rate limiting: maximo 50 mensajes por hora por instancia
  • Deteccion automatica de bots para prevenir abuso
  • Aislamiento multi-tenant: datos, conversaciones y entrenamiento separados por cuenta
  • Backups con encriptacion AES-256 y deteccion de abuso en importacion

6. Contacto

Para consultas sobre privacidad, solicitudes de eliminacion de datos o reportar incidentes de seguridad, contactar al equipo de MIIA a traves del dashboard o por email.
